# This project was made regarding the Empowered BEST Engineering Competition (EBEC Porto '26) 24 hour hackathon. We finished in 🥈***2nd Place***. The problem was the following:
## If meteorology is not fully predictable, can Machine Learning help us understand it better? Can Machine Learning meaningfully support meteorological prediction, or does it become misleading when pushed too far?

### The hackathon was structured into six progressive levels:

- Level 1 – Apply a model to classify whether it's raining or not.
- Level 2 – Apply a model to predict temperature with supervised learning.
- Level 3 – Apply a model to predict snow using unsupervised learning.
- Level 4 –  Apply a model to correlate with incidents on highways. 
- Level 5 – Apply a model to predict meteorological weather values for the next day. .
- Level 6 -  Implement an interface.
- --

### We had access to two different databases in .csv format (one with weather conditions and another with the accidents reports), which were used to learn, validate and test the code. Level 3 worked differently because it was the only one that was unsupervised.

> In order to run the interface, wich is on website format and obviously local-hosted, you need to run the Level2, Level4 and Level5 export functions, to export the ML model and for it to be used on the inteface for the predictions. And the run "streamlit run app.py" on the terminal.
